## 테스트
1. service account, role, role binding 파일 작성으로 pod에 대한 권한 설정 
2. cluster role, cluster role binding 파일 작성으로 node에 대한 권한 설정
3. kubectl 실행이 가능하고 service account 사용하는 pod 정의
4. kubectl 실행이 가능하고 토큰 자동 마운트는 비활성화된 pod 정의 
5. api 직접 호출할 curl 사용 가능 pod 정의 
6. 파일로 클러스터에 설정 적용, 리소스 생성 
7. pod의 권한에 따른 명령 수행 여부 확인 
8. 리소스 정리
